

# 接口文档

本文档详细说明了“酒店评论 API 服务”的所有可用端点，包括酒店查询、评论管理及 AI 摘要功能。

## 1\. 通用说明

* **基础 URL**: `http://localhost:8080`
* **数据交换格式**: `JSON`
* **统一响应结构**:
  ```json
  {
    "code": "200",      
    "data": {},         
    "message": "success"
  }
  ```

-----

## 2\. 酒店管理接口 (Hotels)

### 2.1 获取酒店列表

* **方法**: `GET`
* **路径**: `/api/hotels`
* **说明**: 返回系统中所有酒店的列表（数据来源于种子数据）。

### 2.2 获取酒店详情

* **方法**: `GET`
* **路径**: `/api/hotels/{id}`
* **响应示例**:
  ```json
  {
    "code": "200",
    "data": {
      "id": 1,
      "name": "Grand Palace Hotel",
      "city": "Bangkok",
      "country": "Thailand",
      "stars": 5,
      "createdAt": "2024-01-01T00:00:00"
    },
    "message": "success"
  }
  ```

-----

## 3\. 评论管理接口 (Reviews)

### 3.1 查询指定酒店评论

* **方法**: `GET`
* **路径**: `/api/reviews`
* **参数**: `hotelId` (Long, 必填)
* **说明**: 获取该酒店下的所有历史评论。

### 3.2 新增酒店评论

* **方法**: `POST`
* **路径**: `/api/reviews`
* **请求体**:
  ```json
  {
    "hotelId": 1,
    "reviewer": "Gemini",
    "rating": 5,
    "content": "这里的服务非常出色，AI 响应速度极快！"
  }
  ```

### 3.3 获取 AI 摘要 (核心功能)

* **方法**: `GET`
* **路径**: `/api/reviews/{id}/summary`
* **说明**: 根据预设的提示词模板，对该评论生成一句话摘要。
* **Mock 说明**: 若 `ai.mock.enabled` 为 `true`，将返回模拟生成的摘要文本。

-----

## 4\. 提示词管理接口 (AI Prompts)

用于动态配置 AI 摘要生成的逻辑和风格。

### 4.1 提示词列表

* **方法**: `GET`
* **路径**: `/api/prompts`

### 4.2 保存/切换提示词

* **方法**: `POST`
* **路径**: `/api/prompts`
* **逻辑**: 若 `isActive` 设为 `true`，系统将自动禁用其他所有模板，确保唯一性。
* **请求体**:
  ```json
  {
    "name": "简短风格",
    "content": "你是一个酒店点评专家，请用 10 个字以内总结评论：{{content}}",
    "isActive": true
  }
  ```

-----

## 5\. 错误码定义

| Code | 描述 | 场景举例 |
| :--- | :--- | :--- |
| **200** | 成功 | 正常返回数据 |
| **400** | 业务逻辑异常 | 查询的 ID 不存在、参数缺失 |
| **404** | 资源未找到 | API 路径拼写错误 |
| **500** | 服务器错误 | 数据库连接异常或未预期的运行时错误 |
